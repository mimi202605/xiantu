# creationData 去修仙去战斗化 · 规划文档

> 目标：将 `src/data/creationData.ts` 中的修仙/战斗语境改为通用「天命」语境；**灵根**改为**特质**，并分步转化、记录。

---

## 一、范围与原则

### 1.1 涉及数据块

| 数据块 | 说明 | 转化方向 |
|--------|------|----------|
| LOCAL_WORLDS | 世界列表 | 去修仙/战斗化，保留多元世界观但弱化「修炼、境界、宗门、渡劫」 |
| LOCAL_TALENT_TIERS | 天资品级 | 仙路/仙途 → 天命/资质；废柴/凡人/谪仙等 → 通用表述 |
| LOCAL_ORIGINS | 出身 | 散修/魔道/仙人后裔/夺舍等 → 通用出身（可保留少量奇幻，去掉修仙专有） |
| LOCAL_SPIRIT_ROOTS | 灵根 → **特质** | 金木水火土/灵根/修炼倍率 → 通用特质（性格、天赋倾向、成长加成） |
| LOCAL_TALENTS | 天赋 | 剑道/丹道/天道/灵兽/魔心等 → 通用天赋（能力倾向、社交、成长） |

### 1.2 原则

- **数据结构**：类型名 `SpiritRoot` 可暂时保留（与现有 store/存档兼容），对外展示与文案统一为「特质」。
- **字段语义**：`cultivation_speed` / `base_multiplier` 等保留字段名，仅将**描述与选项内容**改为通用（如「成长加成」「效果强度」）。
- **分步进行**：按上表顺序一部分一部分改，每完成一块在《转化记录》中记一笔。
- **兼容**：存档与代码中已使用 `特质` 与 `灵根` 双读；creationData 只改内容与展示名，不破坏现有读取逻辑。

---

## 二、分步转化计划

### Part 1：灵根 → 特质（LOCAL_SPIRIT_ROOTS）

- **动作**：将当前「灵根」条目全部改为「特质」条目。
- **内容规则**：
  - 名称：不再使用「X灵根」，改为性格/天赋倾向类特质（如「坚韧」「机敏」「亲和」「专注」等）或抽象天赋（如「多面手」「大器晚成」）。
  - 品级：上品/中品/凡品/神品等可保留为 品级/档次，或改为 常见/稀有/传说 等通用档。
  - 描述：去掉「修炼、功法、法术、灵气、天劫」等，改为对性格、处事方式、成长潜力的描述。
  - `cultivation_speed` / `special_effects`：文案改为「成长加成」「特殊效果」等通用表述；数值逻辑可保留供系统使用。
- **导出**：暂保留 `LOCAL_SPIRIT_ROOTS` 导出名，避免全量改引用；在注释和文档中标注为「特质预设数据」。

### Part 2：天资品级（LOCAL_TALENT_TIERS）

- **动作**：仙路/仙途/大道 → 天命/人生/资质；废柴→平庸，凡人→寻常，谪仙转世/大道之子 → 天命所钟等通用表述。
- **原则**：保留「由低到高」的档次与点数逻辑，只改名称与描述文案。

### Part 3：出身（LOCAL_ORIGINS）

- **动作**：散修传人、魔道卧底、仙人后裔、夺舍老怪、药童、阵法学徒、猎妖世家等 → 改为通用出身（市井、书香、商贾、将门、流浪、贵族、手艺传承、神秘身世等）。
- **原则**：`attribute_modifiers` 与 `talent_cost` 可保留；名称与描述去修仙化。

### Part 4：世界（LOCAL_WORLDS）

- **动作**：各世界描述中去掉「灵气、修炼、境界、宗门、渡劫、仙凡」等核心修仙设定；战斗向世界可弱化「战斗」为「冲突、冒险、成长」。
- **原则**：可保留「玄幻/奇幻」氛围，但不对「修仙体系」做硬性依赖；时间与命运感可保留。

### Part 5：天赋（LOCAL_TALENTS）

- **动作**：剑道独尊、丹道圣手、天道眷顾、灵兽亲和、天生魔心等 → 改为通用天赋（如「剑术天赋」「工艺精通」「幸运」「动物亲和」「意志坚定」等）。
- **原则**：`effects` 结构保留；名称与描述去修仙/战斗化，技能名可改为通用（如「剑法」→「剑术」，「炼丹」→「炼药」）。

## 三、与现有代码的衔接

- **类型**：`SpiritRoot` 继续用于特质数据结构（id, name, tier, description, cultivation_speed, special_effects, base_multiplier, talent_cost, rarity）。
- **i18n**：已存在「特质」与「灵根」的切换（USE_MING_PROMPTS）；creationData 改为特质内容后，Ming 模式展示即自然为「特质」。
- **创角页**：Step4 等仍从 `LOCAL_SPIRIT_ROOTS` 拉取选项；内容改为特质后，仅需保证展示文案为「特质」相关键即可。
