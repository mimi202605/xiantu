# 坤舆图 · 地点测试数据集（复杂层级）

用于测试地图嵌套层级、缩放与细化逻辑。数据为**扁平数组**，通过 `上级` 字段在 `locationMapUtils` 中自动建树。

---

## 数据格式说明

每条地点条目（`LocationEntry`）字段：

| 字段       | 类型     | 说明 |
|------------|----------|------|
| `名称`     | string   | 必填，唯一；用于建树与显示。 |
| `描述`     | string   | 可选，悬停/说明用。 |
| `上级`     | string   | 可选，父地点名称；无或不在列表内则视为顶层。 |
| `地点NPC`  | string[] | 可选，该地点内的 NPC 名列表。 |

- **树结构**：由扁平数组按 `上级` 推导，顶层 = 无 `上级` 或 `上级` 不在当前列表中。
- **多根**：可存在多个顶层地点（如多块大陆/多座城），地图上会并排分布。
- **深度与兄弟**：任意层级可有多个子节点，用于压力测试嵌套与缩放一致性。

## 如何导入

数据写入 `世界.信息.地点信息`（即 `worldInfo.地点信息`）。

### 方式一：调试 / 游戏变量面板

若你有「游戏变量」或调试入口可编辑 `世界.信息`：

1. 找到 `世界.信息.地点信息`（或等价路径）。
2. 将下方 **JSON 数组整体** 粘贴为该键的值（替换原有数组）。

### 方式二：从存档 JSON 修改

1. 导出或打开存档 JSON。
2. 定位到 `世界.信息.地点信息`。
3. 用下方数组替换该字段，保存后重新加载存档。

### 方式三：代码里临时写死（开发时）

在初始化世界信息的地方，给 `worldInfo.地点信息` 赋值为下面数组的解析结果即可。

## 建议测试步骤

1. **导入**：用上述方式之一将数组写入 `世界.信息.地点信息`，刷新或重进地图面板。
2. **双根**：确认地图上出现「贝尔塔」与「东荒大陆」两个顶层框。
3. **缩放与细化**：  
   - 放大并双击「贝尔塔」→ 应进入贝尔塔细化，上城区/下城区/港口/总部等并列可见。  
   - 再放大并双击「贝尔塔·上城区」→ 第二层保持大框，不缩小；苏家庄园、林氏公馆、议会厅等可见。  
   - 再双击「贝尔塔·上城区·苏家庄园」→ 第三层保持，书房与大厅可见。  
4. **东荒侧**：对「东荒大陆」→「青云域」→「天剑宗」做同样逐层放大，检查每层比例是否一致、无上层突然缩小。
5. **Tooltip**：悬停「贝尔塔·上城区·苏家庄园·书房」，应能看到描述与「在场：苏天成」（若关系数据中有该 NPC）。

若 2～4 层展开时上层框保持预期大小、不缩回小尺度，则嵌套与细化逻辑通过本数据集验证通过。
